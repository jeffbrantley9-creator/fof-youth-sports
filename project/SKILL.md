If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. Always link or import `colors_and_type.css` for tokens, and reuse the SVG logos and pattern files from `assets/`. Use Phosphor icons for sport pictograms and Lucide icons for general UI (both via CDN — see README's Iconography section).

If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand: navy + gold palette, Nunito (display) + DM Sans (body), pill-shaped CTAs with stacked gold shadows, sentence-case copy, no emoji, real-photo imagery (warm/sunlit/candid).

If the user invokes this skill without any other guidance, ask them what they want to build or design (a new sport's landing page, a coach onboarding flow, a parent email template, a sponsor pitch deck, etc.), ask a few questions about audience and scope, and act as an expert designer who outputs HTML artifacts or production code, depending on the need.
